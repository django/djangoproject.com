{% translate "Django Code of Conduct - Changes" %}

{% translate "Change control process" %}

{% blocktranslate trimmed %} We're (mostly) programmers, so we'll track changes to the code of conduct and associated documents the same way we track changes to code. All changes will be proposed via a pull request to the Django Code of Conduct repository on GitHub. Changes will be reviewed by the code of conduct working group first, and then sent to the DSF and the Django community for comment. We'll hold a comment period of at least one month. Changes to the Code of Conduct itself will be voted on by the DSF board; other documentation changes will be approved by the code of conduct working group. Approved changes will be merged, published, and noted in our changelog.{% endblocktranslate %}

{% blocktranslate trimmed %} This only applies to material changes; changes that don't affect the intent (typo fixes, re-wordings, etc.) can be made immediately{% endblocktranslate %}

{% blocktranslate trimmed %} A complete list of changes can always be found on GitHub.{% endblocktranslate %}
